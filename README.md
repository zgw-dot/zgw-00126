# 课程补考资格与安排管理系统

一个面向学生、任课教师和教务管理员的课程补考资格与安排管理系统。

## 功能特性

- **成绩管理**：支持批量导入课程成绩 CSV，自动计算补考资格
- **资格管理**：资格来源可追溯（自动计算 / 人工覆盖），历史记录不可删除
- **申请流程**：学生在线提交补考申请，教务审核批准或拒绝
- **排考管理**：自动检测考试时间冲突，考场座位实时扣减与释放
- **数据导出**：补考通知名单、考试安排表一键导出 CSV
- **数据持久化**：重启后资格结果、审核历史、阈值配置、考场余量保持一致
- **三级权限**：学生 / 教师 / 教务，角色权限隔离

## 技术栈

- **前端**：React 18 + TypeScript + Vite 6 + Tailwind CSS 3 + Zustand
- **后端**：Express 4 + TypeScript + sql.js（WASM SQLite）
- **部署**：前后端分离，Vite 代理 API 请求

## 核心业务流程

### 完整补考流程

1. **教务导入成绩**：教务管理员在「成绩管理」页面上传成绩 CSV
2. **自动计算资格**：系统根据阈值（默认 60 分）自动计算补考资格
3. **学生查看资格**：学生登录后在「补考资格」页面查看有资格的课程
4. **学生提交申请**：学生选择课程提交补考申请
5. **教务审核申请**：教务在「申请审核」页面批准或拒绝申请
6. **教务安排考试**：教务在「考试编排」页面为已批准的申请安排考场和时间
7. **查看考试安排**：学生和教师可在「考试安排」页面查看排考结果
8. **导出数据**：教务可导出补考通知名单和考试安排表

### 资格人工覆盖

- 教务可在「资格管理」页面为学生人工授予或取消补考资格
- 覆盖时必须填写原因
- 原资格记录保留（标记为已取消），新记录标记为「人工覆盖」来源
- 所有操作记录审计日志

### 取消资格与级联操作

- 取消资格时，关联的申请会被级联拒绝
- 已排考的会被级联取消，座位自动释放
- 取消原因会记录在审计日志中

## 失败路径说明

### 1. 不达标学生申请补考

**现象**：成绩高于及格线（无补考资格）的学生尝试提交申请

**结果**：系统拒绝申请，提示「没有该课程的补考资格」

**验证方法**：
- 用 student1 登录，尝试为成绩 80 分以上的课程提交申请
- 或直接调用 POST /api/applications，传入无资格的课程 ID

### 2. 同一学生考试时间冲突

**现象**：为同一学生安排两门时间重叠的考试

**结果**：系统拒绝排考，提示「存在时间冲突」

**验证方法**：
- 先为学生安排一门考试（如 2024-06-15 09:00-11:00）
- 再尝试为同一学生安排同一时间段的另一门考试
- 系统会检测到冲突并拒绝

### 3. 取消已批准申请后座位未释放

**现象**：取消已排考的申请后，考场座位数未恢复

**预期**：取消排考（或取消资格、拒绝申请）后，对应考场的已用座位数实时减少

**验证方法**：
1. 记录考场当前已用座位数
2. 安排一名学生到该考场考试，确认座位数 +1
3. 取消该排考记录，确认座位数 -1
4. 或拒绝申请 / 取消资格，确认座位数同步释放

### 4. 教师越权审批

**现象**：教师角色尝试审批申请或修改资格

**结果**：系统返回 403 Forbidden，提示「权限不足」

**验证方法**：
- 用 teacher1 登录，尝试访问教务管理页面（会被路由守卫拦截）
- 或直接调用 POST /api/applications/:id/approve，使用教师 token
- 系统返回 403

### 5. 成绩导入格式错误

**现象**：上传的 CSV 缺少必要字段或格式错误

**结果**：系统返回 400，提示具体错误信息

### 6. 考场容量不足

**现象**：排考时考场剩余座位不够

**结果**：系统拒绝排考，提示「考场容量不足」

## 数据持久化

系统使用 sql.js（SQLite 的 WebAssembly 版本），数据自动保存到 `data/db.sqlite` 文件。

- 重启服务后，所有数据保持不变
- 包括：用户、成绩、资格、申请、考场、排考、阈值配置、审计日志
- 首次启动时自动创建数据库表和种子数据

| POST | /api/applications/:id/approve | 批准申请 | 教务 |

| DELETE | /api/applications/:id | 撤回申请 | 学生 |
| GET | /api/exam-rooms | 考场列表 | 全部 |

## 开发说明

### 数据库重置

如需重置数据库，删除 `data/db.sqlite` 文件后重启服务即可。

### 阈值配置

- 默认及格阈值：60 分
- 修改阈值后自动重新计算所有学生的补考资格
- 阈值变更记录在 `threshold_history` 表中
